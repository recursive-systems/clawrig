# 2026-03-11

## [AM] Async test flakiness: global state mutation in ExUnit

**Context:** `updater_test.exs` was `async: true` but used `System.put_env("HOME", ...)` in setup. Race condition with other async tests — passed in isolation, failed with specific seeds when another test's `on_exit` restored `HOME` mid-assertion.

**Decision:** Made `CodexAuth.auth_path` configurable via `Application.get_env(:clawrig, :codex_auth_path, ...)`, matching `OpenAICredentials.auth_profiles_path`. Test sets app env instead of mutating global `HOME`.

**Alternatives considered:** Setting `async: false` on the test module — rejected because it would slow down the suite and mask the real design issue.

**Implications:** Any ClawRig module reading `HOME` or other global env vars should use app-env-configurable paths so tests stay `async: true`. Pattern: `Application.get_env(:clawrig, :some_path, <default from HOME>)`. Audit candidates: any module calling `System.get_env("HOME")` directly.
